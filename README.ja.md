# セキュアなゲノムデータ環境におけるローカルLLMコーディング戦略

> **[English version (README.md)](README.md)**

## 課題

患者由来の機密ゲノムデータを扱うセキュア環境において、AIによるコーディング支援を導入したい。想定するデプロイ環境は2つ:

1. **セキュアサーバー**（SSH/VPN経由）: インバウンド通信（ソフトウェアのダウンロード）は許可、アウトバウンド通信は禁止。直接インストール可能。
2. **院内環境**（Windows 11）: インバウンド・アウトバウンドともに原則禁止。USBメモリ等の可搬媒体でソフトウェアを持ち込み、ウイルススキャン後に利用可能。

## 調査内容

### Claude Codeはローカルモデルで動作するか？

Claude Codeのバックエンドをローカルモデル（Qwen 3.5 + Ollama）に置き換える方法を検証した。**結論: 主要な手段としては非推奨。** Claude CodeはAnthropicのClaudeモデルに強く依存しており、エージェントループ、ツール呼び出し、拡張思考、プロンプトキャッシュなどがClaude固有の機能として実装されている。コミュニティによる回避策（Ollama v0.14+のAnthropic API互換、local-claude-codeプロジェクト、LiteLLMプロキシ）は存在するが、動作が不安定でサポートもない。

### 代替ツール

任意のモデルバックエンドに対応したローカルファーストのコーディングアシスタントを評価した:

- **Aider**（推奨） — 開発歴3年の成熟したツール。AIの編集を自動でgit commitする監査証跡機能、`/ask`モード（コード探索）、`/architect`モード（指示に基づくコード生成）、編集後の自動テスト・lint実行
- **OpenCode**（代替案） — リッチなTUI、Go製シングルバイナリ（持ち込みが容易）、エアギャップモード対応、カスタムエージェント機能

### ローカルモデルの選定

**Qwen 3.5** — Apache 2.0ライセンス、397Bパラメータ（MoEにより実行時17B）、SWE-bench Verified 72.4（27BモデルはGPT-5 miniと同等）、優れたツール使用ベンチマーク。0.6Bから397Bまで複数サイズあり。

## 解決策: ハイブリッドワークフロー

ローカルの小型モデルだけに頼るのではなく、各モデルの得意領域を活かす**ハイブリッド方式**を採用する:

```
セキュア環境（SSH）                  外部（手元のPC）
====================                ================

1. ローカルエージェントで探索  ──>    2. Claudeで設計・計画
   （Aider /ask + Qwen 3.5）           （メタデータを渡し、計画を受け取る）
   収集: スキーマ、ファイル構成、
   カラム名、ツールバージョン等

3. ローカルエージェントで実行  <──    Claudeがステップごとの
   （Aider /architect + Qwen 3.5）    実装指示を返す

4. 自動テスト・lint
   単純なエラー: ローカルで修正
   複雑なエラー: ──────────── ──>    5. Claudeでデバッグ
```

人間のオペレーターが**データ境界のチェックポイント**として機能する。境界を越えるのは非機密のメタデータ（ファイルスキーマ、ディレクトリ構成、コード構造、エラーメッセージ）のみ。ゲノムデータは絶対に環境外に出さない。

## クイックスタート

### シナリオA: セキュアサーバー（インバウンド通信可能）

```bash
./install.sh                          # GPU自動検出、デフォルトで27Bモデル
./install.sh --model 9b               # モデルサイズを指定
./install.sh --cpu                    # CPUのみモード（9Bにフォールバック）
```

### シナリオB: 院内エアギャップ環境（ネットワーク不可）

```bash
# 1. インターネット接続可能なPCで、必要なソフトウェアを一括ダウンロード:
./download.sh                              # デフォルト: Windows 11向け、9bモデル
./download.sh --model 27b                  # より大きなモデル
./download.sh --os linux                   # Linux向けにする場合

# 2. 出力ディレクトリをUSBメモリにコピーし、院内ポリシーに従いウイルススキャン

# 3. 院内PC上で実行（Windows PowerShell、管理者権限）:
.\install-offline.ps1
```

### コーディング開始（両シナリオ共通）

```bash
ollama serve &
aider --model ollama/qwen3.5:9b      # ローカルモデルでAiderを起動

# Aider REPL内で以下のコマンドを使用:
#   /ask describe the project structure    （読み取り専用の探索）
#   /architect                             （architectモードに切替）
#   Claudeの計画をここに貼り付け            （計画を実行）
```

## 主要な意思決定

| 判断 | 根拠 |
|------|------|
| 完全ローカルではなくハイブリッドワークフロー | Claudeの推論能力は設計・アーキテクチャ策定において圧倒的に優れている。ローカルモデルは詳細な指示に従うだけでよい |
| Claude CodeではなくAider | 任意のモデルに対応する設計。git自動コミットにより監査証跡を確保。`/ask` + `/architect`がワークフローに直接対応 |
| 他のローカルモデルではなくQwen 3.5 | オープンウェイトモデルの中でコード生成・ツール使用のベンチマークが最高水準。Apache 2.0ライセンス。柔軟なサイズ展開 |
| vLLMではなくOllama | シングルユーザー環境に最適なシンプルさ。Anthropic API互換。インストールが容易 |
| 人間によるデータ境界チェック | 組織の要件。自動化されたデータ転送は存在せず、人間が目視で確認することで意図しないデータ漏洩を防止 |
